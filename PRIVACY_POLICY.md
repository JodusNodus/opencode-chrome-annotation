# Privacy Policy for OpenCode Chrome Annotation

Effective date: 2026-04-29

OpenCode Chrome Annotation helps users send browser annotation context into OpenCode.

## What this extension does

When you use the extension, it can capture and send annotation context that you explicitly trigger:
- Your annotation text/comment
- Selected element metadata (for example selector, tag, element text, position)
- Current page context (for example page URL and title)
- Screenshot data captured when you submit an annotation

## How data is used

Data is used only to deliver your annotation context to your OpenCode workflow so OpenCode can act on your request.

The extension communicates with a local plugin instance running on your machine (localhost).

The extension does not transmit annotation payloads to any extension-controlled remote backend.
Annotation data is delivered only to your own local OpenCode setup.

## Data sharing and sale

- We do not sell your data.
- We do not use your annotation data for advertising.
- We do not use third-party tracking SDKs in the extension.

## Data storage

- Annotation screenshot files may be written to a local temporary directory on your machine by the plugin runtime.
- The extension itself does not operate a remote backend service for annotation uploads.

## User control

- Annotation capture is user-initiated.
- You choose when to connect a tab and when to submit an annotation.
- You can remove the extension at any time from `chrome://extensions`.

## Security

The extension communicates with localhost endpoints to reach your local OpenCode plugin instance.

No remote transport is required for the annotation flow implemented by this extension.

## Changes to this policy

We may update this policy over time. Material changes will be reflected by updating the effective date.
